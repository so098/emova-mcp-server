최근 코드 변경사항을 분석하여 README.md에 반영해야 할 내용이 있는지 확인하고, 필요하면 수정합니다.

## 워크플로

### 1. 변경사항 파악

아래 명령어로 최근 변경사항을 확인하세요:

```bash
# 커밋되지 않은 변경사항이 있으면
git diff HEAD

# main 브랜치 대비 변경사항이 있으면
git diff main

# 최근 커밋 변경사항
git log --oneline -5
git diff HEAD~3
```

변경된 파일 목록과 diff 내용을 모두 확인하세요.

### 2. README.md 현재 내용 읽기

`README.md` 파일을 읽어 현재 문서화된 내용을 파악하세요.

### 3. 업데이트 필요 여부 판별

변경사항 중 아래 항목에 해당하는 것이 있는지 확인하세요:

- **새 도구(Tool) / 리소스(Resource) / 프롬프트(Prompt) 추가**: 구성 요소 테이블에 반영 필요
- **아키텍처 변경**: 디렉토리 구조, 새 파일/모듈 추가 시 구조도 업데이트
- **새 npm 스크립트 추가**: 빌드 & 테스트 섹션 업데이트
- **새 의존성 추가**: 의존성 섹션 업데이트
- **빌드/실행 방법 변경**: 설치/실행 가이드 업데이트
- **새 타입/가설/감정 카테고리 추가**: 도메인 지식 섹션 업데이트
- **환경 변수 추가**: 설정 섹션 업데이트

### 4. 결과 처리

- **업데이트 필요**: README.md를 수정하고, 어떤 섹션을 왜 변경했는지 요약 보고
- **업데이트 불필요**: "README.md 업데이트 불필요: 변경사항이 문서에 영향을 주지 않습니다." 로 보고
